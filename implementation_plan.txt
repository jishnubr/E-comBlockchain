# E-comBlockchain Modernization Plan

This plan details the migration of the E-comBlockchain project from a legacy SSR Monolith (Servlets/JSPs) to a modern, decoupled architecture using **Spring Boot 4.0.0 (Java 25 LTS)** and **React (Vite)**.

## User Review Required

> [!IMPORTANT]
> - **Java 25 LTS & Spring Boot 4**: We are targeting a future stack. Ensure the environment supports these versions (or we will use the latest stable available).
> - **JWT Authentication**: We are moving from session cookies to stateless JWT tokens.

## Proposed Changes

### Backend (Spring Boot 4)

Summary of the new backend structure using Java 25 features.

#### [NEW] `com.ecomblockchain.dto` [DTOs as Java 25 Records]
Using Records for immutable data transfer between the API and React.

#### [NEW] `com.ecomblockchain.config.SecurityConfig` [Spring Security 7.0]
Stateless configuration for JWT authentication.

#### [NEW] `com.ecomblockchain.controller.SmartContractController`
REST endpoints for smart contract interaction using Pattern Matching for Switch.

## Core Engine Implementation (Phase 2)

To restore full functionality from the legacy JDBC Servlets to Spring Boot, we will implement the following:

### 1. Data Layer Expansion (`com.ecomblockchain.model`)
- **Modify [User.java](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/backend/src/main/java/com/ecomblockchain/model/User.java)**: Add `balance` (BigDecimal) and `signatureData` (byte[]).
- **New Entities & Repositories**: Create `Product`, [Order](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/backend/src/main/java/com/ecomblockchain/dto/OrderDto.java#5-11), [Block](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/backend/src/main/java/com/ecomblockchain/EcomBlockchainApplication.java#6-12), [Blockchain](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/backend/src/main/java/com/ecomblockchain/EcomBlockchainApplication.java#6-12), `Node`, [Contract](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/backend/src/main/java/com/ecomblockchain/controller/SmartContractController.java#20-40), and `CentralAccount` entities with corresponding SD-JPA Repositories.

### 2. Cryptography Engine (`com.ecomblockchain.service`)
- **[NEW] `CryptoService`**: Port the legacy `Generator.java` logic. Implement `generateHash(byte[])`, `generateECDSA(String)`, and `verifySignature(String, String, String)`.

### 3. Smart Contract & Escrow Logic
- **Modify [ContractService](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/backend/src/main/java/com/ecomblockchain/service/ContractService.java#7-26)**: Rewrite logic to use `@Transactional` database operations instead of mocked strings.
- Implement the Escrow process: Deduct collateral from Buyer `balance` -> transfer to `CentralAccount`.
- Implement 3-Strike Verification Rule and Collateral Redistribution logic.

### 4. Frontend Dashboards 
- **[NEW] `SellerDashboard.jsx`**: Forms to POST new products to `/api/v1/products` and view incoming orders.
- **[NEW] `TransporterDashboard.jsx`**: UI to bid on shipments and update geographic `curr_location`.
- **[NEW] `AdminDashboard.jsx`**: Global transaction overview and Arbitrator contract-signing capabilities.
- **Route Protection**: Update [App.jsx](file:///c:/Users/JISHNU%20B%20R/Git%20projects/E-comBlockchain/frontend/src/App.jsx) with strict role-based `<Route>` definitions.
